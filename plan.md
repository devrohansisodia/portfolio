## 9. Data-driven Approach

Use separate data files so content can be updated easily without editing every component.

Example:

```txt
src/data/profile.js
src/data/skills.js
src/data/projects.js
src/data/experience.js
```

This keeps code clean and GitHub-friendly.

## 10. Theme Mode Plan

Dark / light mode should be implemented using Tailwind `dark` class strategy.

Requirements:

- Add dark mode support in `tailwind.config.js`
- Use `localStorage` to save theme preference
- Default theme:
  - Use saved preference if available
  - Otherwise use system preference
- Add toggle button in navbar
- Smooth color transition between themes
